# 1:systemdの概要と操作
systemdではUnitと呼ばれる処理単位を使い、OSの動作モードやサービスの状態などの管理を行う。

- service 各種サービスの起動・停止を管理
- target 複数のUnitをグループ化して管理
- socket プロセス間の通信ソケットを管理
- timer タイマー管理

# 2:ターゲット
SysVinitではOSの動作モードをランレベルで表現するが、systemdではターゲットと呼ぶ。

- default.target    起動直後のOSの動作モード
- poweroff.target   システム停止状態
- rescue.target     rootユーザーのみでログインできる状態
- multi-user.target 複数の一般ユーザーがログインできる状態(CUI)
- graphical.target  複数の一般ユーザーがログインできる状態(GUI)
- reboot.target     再起動の状態
- emergency.target  可能な限り最小限の環境で起動し、システムを修復するためにrootユーザーのみログインできる状態

systemctlコマンドで変更が可能
systemctl サブコマンド 対象Unit
サブコマンド一覧
- get-default            default.targetファイルがどのターゲットにリンクしているかを表示
- set-default ターゲット名 default.targetファイルのリンク先の変更
- isolate ターゲット名     今すぐ指定したターゲットへ変更

# 4:ターゲットの変更を伴わないサービスの起動・停止
start      指定のサービスを即時起動
stop       指定のサービスを即時停止
restart    即時再起動
status     現在のステータスを表示
is-active  現在起動しているかどうか
enable     linux起動時に指定のサービスを起動する
disable    linux起動時に指定のサービスを起動しないように
is-enabled 指定のサービスがlinux起動時に自動起動するかどうか
