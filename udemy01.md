# 4
個別パーティションは/dev/sda1, /dev/sda2と続いていく。

# 5
/bootディレクトリに含まれる重要なファイルの理解。
このディレクトリはLinuxシステムが起動するために必要なファイルを保持していて、システム起動プロセスの初期段階で重要な役割を果たす。

Linux kernel images
カーネルイメージはシステム起動時に読み込まれ、カーネルをメモリにロードするために使用される。

Initial ramdisk images
ブートプロセス中にルートファイルシステムをマウントする前に必要な一時的なファイルシステムを提供する。

# 8
コンテナ：軽量な仮想化手法で、ホストOSのカーネルを共有しながら独立したプロセス空間を提供。迅速なデプロイやスケーラビリティに優れている。
アプリケーションの隔離とスケーラビリティに優れている。

仮想マシン：ハイパーバイザーによって物理ハードウェアを仮想化し、完全な仮想環境を提供。異なるOSを実行できるがオーバーヘッドが大きくリソース消費も多い。
異なるOSを動作させる必要がある場面に向く。
