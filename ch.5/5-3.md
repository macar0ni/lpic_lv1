# シェルスクリプトの基本ルール
シェルスクリプトとは、シェルで実行できる一連のコマンドをファイルに記述した実行ファイル。  

書き方の推奨ルール
・ファイル名に拡張子.shを付ける。絶対ではないが一般的。   
・シェルスクリプト1行目に#!で始まる行を記述する。  

# シェルスクリプトの実行方法
ログインシェルに設定しているシェル変数を参照せずにシェルスクリプトを実行したい場合はbashコマンド。

現在起動中のシェルに設定されているシェル変数やエイリアスを参照しながらシェルスクリプトを実行したいのであれば、sourceコマンドまたは.コマンド。

ファイルパスを指定して実行。


スクリプトファイル名のみで実行。

# 3:特殊な変数
- $0 スクリプトファイル名
- $1 1つめの引数を表す。２つめなら$2。
- $@ すべての引数。
- $# 引数の個数
- $$ 現在利用しているシェルのPID。
- $? 終了ステータスを格納。0は成功、0以外は何らかのエラーで終了。

# 4:制御構造
条件分岐処理、ifコマンドを使用。
~~~
if 条件式
then
 コマンド1
else
 コマンド1
fi
~~~

ifコマンドの条件式の部分にはtestコマンドを使用する場合がある。
testコマンドは条件式を評価して真偽をチェックし、真の場合は0を、そうでない場合は0以外を返す。
~~~
test 条件式
もしくは
[ 条件式 ]
~~~

繰り返し処理のwhile。
~~~
while 条件式
do
 コマンド
done
~~~

readコマンドは標準入力から値を読み込み、引数に指定されている変数に格納する。

seqコマンド
1から終了値まで1ずつ増やして出力。
~~~
seq 終了値
~~~
開始値から終了値まで1ずつ増やして出力。
~~~
seq 開始値 終了値
~~~
開始値から終了値まで間隔値ずつ増やして出力。
~~~
seq 開始値 間隔値 終了値
~~~

forコマンド。
~~~
for 変数名 in 変数に代入する値のリスト
do
 コマンド
done
~~~

functionコマンド
~~~
function 
 関数名()
 {
  コマンド
 }
~~~
