# ファイルシステムとは
ファイルやディレクトリを管理できるようにする仕組み。

種類
・ext2    linux標準ファイルシステム
・ext3    ext2にジャーナリング機能（ファイルシステム操作の記録機能）を加えたファイルシステム
・ext4    ext3を機能拡張したファイルシステム
・XFS     高度な機能を持つファイルシステム、大容量ファイルが扱える
・Btrfs   linux向けの先進的なファイルシステムで、raidやスナップショットの機能を持つ
・VFAT    主にUSBメモリやSDカードで使用されるファイルシステム
・exFAT   過去のwindowsで使用されていたファイルシステムであるFATを機能拡張したもの
・iso9660 CDやDVDで使用されるファイルシステム

# mkfs
mkfs -t タイプ デバイスファイル名
パーティションにファイルシステムを作成する。

ext2だったら
mkfs -t ext2 デバイスファイル名

# mke2fs
mke2fs デバイスファイル名
ext2/3/4ファイルシステムを作成。

-j
ext3ファイルシステムを作成

-c
実行前にパーティションに不良がないかをチェック

# fsck
fsck デバイスファイル名

ファイルシステムの状態をチェックして破損や不整合を修復

# e2fsck
e2fsck デバイスファイル名
ext2/3/4ファイルシステムのチェックと修復

# xfs_repair
xfs_repair デバイスファイル名
XFSファイルシステムのチェックと修復

XFSとは大容量ファイルを扱えるなど、高度な機能を持つファイルシステム

# dumpe2fs
dumpe2fs デバイスファイル名
ext/2/3/4ファイルシステムの管理情報の「確認」

# tune2fs
tune2fs デバイスファイル名
ext/2/3/4ファイルシステムの管理情報の「編集」

ext2 linux標準ファイルシステム
ext3 2にジャーナリング機能を加えた
ext4 3を機能拡張

# xfs_info
XFSファイルシステムの管理情報の確認

# xfs_admin
XFSファイルシステムの管理情報の編集

XFSとは高度な機能を持つファイルシステム

# Debian形式のパッケージ「apache2」のインストール作業が予期せず中断され、不完全な状態となった。
# 依存関係の解決を含めてシステムを修復し、インストールを完了させるためにまず実行すべき適切なコマンドは
-fまたは--fix-broken install
すでにシステムに存在する、依存関係の問題や不完全なパッケージを修復

# yumコマンドを使用して、アップデート可能な全てのパッケージをアップデートしたい。
yum update
yumのupdateサブコマンドに引数を設定しない場合、すべてのパッケージをアップデートする

# apt-getコマンドを使用して、パッケージの削除を行わずに、インストール済みの全パッケージを対象として更新を実行したい。
apt-getのupgradeとdist-upgradeのちがい
upgradeは、
新しいパッケージの追加や、既存パッケージの削除が必要な場合は更新しない。
dist-upgradeはする。

# apt-cacheコマンドを使用して、「ssh」パッケージへ依存しているパッケージを含めた詳細情報を表示させたい。
search  キーワードを含むパッケージを検索、パッケージを検索！！
show    指定したパッケージの詳細
showpkg 指定したパッケージの被依存関係を含めた情報
depends 指定したパッケージが依存しているパッケージの一覧

# zypperコマンドを使用して、アップデート可能なパッケージを確認したい。
list-updates もしくは lu
updates sが付く！！

# zypperコマンドを使用して、リポジトリの一覧を表示させたい。
リポジトリの一覧表示はzypper reposか、zypper lr

# zypperコマンドを使用して、リポジトリを更新したい。
リポジトリの更新はrefresh

# dpkgの設定ファイルは/etc/dpkg/dpkg.cfg
